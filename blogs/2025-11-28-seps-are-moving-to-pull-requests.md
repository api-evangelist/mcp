We’re updating how Specification Enhancement Proposals (SEPs) are submitted and managed. Starting today, SEPs will be created as pull requests to the seps/ directory instead of GitHub issues. Why the Change? When we introduced SEPs in July , we chose GitHub Issues as our starting point. Issues are familiar to developers, low-friction, and got us up and running quickly. But as more proposals have come through the process, we’ve identified some key pain points: Scattered discussions. With issues, the proposal text lives in the issue body while implementation details often end up in a separate PR
